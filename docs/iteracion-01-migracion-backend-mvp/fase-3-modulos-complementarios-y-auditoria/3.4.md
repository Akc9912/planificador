# 3.4 Implementar Audit Module

## Objetivo

Implementar el modulo `audit` para registrar acciones criticas de la aplicacion a nivel servicio usando AOP o un mecanismo equivalente, sin depender de triggers de DB como unica estrategia.

## Por que se hace

El sistema necesita trazabilidad de cambios sensibles. Si la auditoria se resuelve con logs informales o duplicando codigo en cada servicio, se vuelve incompleta y dificil de mantener.

## Pasos de ejecucion

1. Confirmar dependencia necesaria para AOP o definir alternativa equivalente.
2. Crear estructura `modules/audit`.
3. Modelar `audit_logs` y DTOs de consulta.
4. Implementar anotacion o mecanismo central de auditoria.
5. Registrar accion, usuario, tabla, payload y metadatos disponibles.
6. Exponer endpoints de consulta y agregar pruebas.

## Comandos exactos

```bash
grep -n "spring-boot-starter-aop\|aspectj" pom.xml
find src/main/java/aktech/planificador/modules -path "*/audit/*" | sort
grep -R "@Aspect\|Auditable\|audit" -n src/main/java/aktech/planificador
./mvnw -q -DskipTests compile
./mvnw -q -DskipTests=false -Dtest=Audit*Test,ModuleBoundariesTest test
```

## Ejemplo de codigo

```java
@Aspect
@Component
public class AuditAspect {

    @AfterReturning("@annotation(Auditable)")
    public void logAction(JoinPoint joinPoint) {
        auditService.register(joinPoint.getSignature().getName());
    }
}
```

## Verificacion

- Existe modulo `audit` y mecanismo centralizado de captura.
- Las acciones criticas generan registros consultables.
- La auditoria no obliga a ensuciar controladores o servicios con codigo repetido.
- Existen pruebas del modulo.

## Resultado esperado

El backend gana trazabilidad real sobre operaciones sensibles y deja base para soporte, debugging y seguridad operativa.

## Criterio de cierre

- [ ] Existe modulo `audit` con estructura completa.
- [ ] Hay mecanismo centralizado de auditoria.
- [ ] Existen endpoints de consulta.
- [ ] Las pruebas del modulo existen y pasan.

## Troubleshooting breve

Si la estrategia AOP no alcanza para un caso especifico, documentar la excepcion y agregar la captura en un punto unico del servicio. No duplicar logging manual por todas partes.
