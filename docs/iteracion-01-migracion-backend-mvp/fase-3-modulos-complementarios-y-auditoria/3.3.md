# 3.3 Implementar UserSettings Module

## Objetivo

Implementar el modulo `user-settings` para gestionar configuracion personalizada del usuario con defaults, ownership y operacion idempotente tipo upsert.

## Por que se hace

Las preferencias del usuario no deben quedar mezcladas con el modelo legacy ni resueltas por configuracion ad hoc del frontend. Necesitan un modulo simple, claro y consistente.

## Pasos de ejecucion

1. Crear estructura del modulo de configuracion.
2. Modelar `user_settings` con `user_id` como referencia externa.
3. Implementar endpoint `GET`, `PUT` y `POST /reset`.
4. Definir defaults al primer acceso.
5. Validar rangos y coherencia de horarios del planner.
6. Agregar pruebas del modulo.

## Comandos exactos

```bash
find src/main/java/aktech/planificador/modules -path "*/user*settings*/*" | sort
grep -R "user_settings\|planner_start_hour\|planner_end_hour\|settings" -n src/main/java/aktech/planificador src/main/resources
./mvnw -q -DskipTests compile
./mvnw -q -DskipTests=false -Dtest=UserSettings*Test,ModuleBoundariesTest test
```

## Ejemplo de codigo

```java
public UserSettingsResponseDto getOrCreate(UUID userId) {
    return userSettingsRepository.findByUserId(userId)
            .map(mapper::toResponse)
            .orElseGet(() -> createDefaults(userId));
}
```

## Verificacion

- Existe modulo nuevo con endpoints de lectura, actualizacion y reset.
- Los defaults se crean o aplican al primer acceso.
- El usuario solo opera su propia configuracion.
- Existen pruebas del modulo.

## Resultado esperado

El backend puede devolver y persistir preferencias del usuario de manera predecible, sin delegar esa responsabilidad a logica cliente o legacy.

## Criterio de cierre

- [ ] Existe modulo de configuracion con estructura completa.
- [ ] Hay defaults documentados y aplicados.
- [ ] Hay validaciones de rango y coherencia.
- [ ] Las pruebas del modulo existen y pasan.

## Troubleshooting breve

Si una preferencia no existe todavia en DB, devolver valor efectivo por default. No forzar a frontend a adivinar defaults locales.
