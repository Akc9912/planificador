# 3.2 Implementar Reminder Module

## Objetivo

Crear el modulo `reminder` para gestionar recordatorios, relaciones con materias y base de futuras notificaciones sin depender del flujo legacy hoy desactivado.

## Por que se hace

Reminder comparte el mismo problema que Event: hoy esta fuera del MVP y desacoplado via `410 GONE`. La vuelta correcta requiere un modulo nuevo, ownership claro y relaciones controladas.

## Pasos de ejecucion

1. Crear estructura `modules/reminder`.
2. Modelar `reminders` y `reminder_subjects` con UUID.
3. Implementar CRUD, filtros por materia y vinculaciones.
4. Validar ownership en cada operacion.
5. Dejar preparada la base para notificaciones futuras sin implementarlas todavia.
6. Crear pruebas del modulo.

## Comandos exactos

```bash
grep -R "RecordatorioController\|410 GONE\|Reminder" -n src/main/java/aktech/planificador
find src/main/java/aktech/planificador/modules -path "*/reminder/*" | sort
./mvnw -q -DskipTests compile
./mvnw -q -DskipTests=false -Dtest=Reminder*Test,ModuleBoundariesTest test
```

## Ejemplo de codigo

```java
public void linkSubject(UUID reminderId, UUID subjectId, UUID userId) {
    subjectApi.assertUserOwnsSubject(userId, subjectId);
    reminderSubjectRepository.save(new ReminderSubject(reminderId, subjectId));
}
```

## Verificacion

- Existe estructura modular del modulo Reminder.
- Los recordatorios pueden vincularse a materias validando ownership.
- No se depende de codigo legacy para devolver o guardar recordatorios.
- Hay pruebas del modulo y el guardrail sigue verde.

## Resultado esperado

Reminder queda reincorporado como modulo modular, listo para crecer luego hacia notificaciones o agendas sin deudas de acoplamiento.

## Criterio de cierre

- [ ] Existe modulo `reminder` con capas completas.
- [ ] CRUD y relaciones con materias funcionan.
- [ ] Ownership validado en operaciones sensibles.
- [ ] Las pruebas del modulo existen y pasan.
- [ ] El legacy sigue fuera del flujo nuevo.

## Troubleshooting breve

Si Reminder necesita datos de Subject, exponerlos por contrato compartido. No copiar repositorios ni importar servicios de otro modulo.
