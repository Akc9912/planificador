# 3.1 Implementar Event Module

## Objetivo

Crear el modulo `event` para gestionar eventos academicos, sus horarios y su relacion con materias, reemplazando la desactivacion actual por una implementacion modular real.

## Por que se hace

Hoy Event esta fuera del MVP y sus endpoints legacy fueron desacoplados a `410 GONE`. Reintroducirlo bien exige un modulo nuevo, no volver a habilitar controladores legacy.

## Pasos de ejecucion

1. Confirmar que la reintroduccion de Event esta realmente habilitada por roadmap.
2. Crear estructura `modules/event` con capas completas.
3. Modelar `events`, `event_schedules` y `event_subjects` con UUID.
4. Implementar CRUD y filtros por rango y materia.
5. Resolver vinculaciones con Subject usando contratos compartidos.
6. Crear pruebas del modulo y dejar `ModuleBoundariesTest` en verde.

## Comandos exactos

```bash
grep -R "EventoController\|410 GONE\|Event" -n src/main/java/aktech/planificador
find src/main/java/aktech/planificador/modules -path "*/event/*" | sort
./mvnw -q -DskipTests compile
./mvnw -q -DskipTests=false -Dtest=Event*Test,ModuleBoundariesTest test
```

## Ejemplo de codigo

```java
@GetMapping("/api/events")
public ResponseEntity<List<EventResponseDto>> getEvents(
        @RequestParam(required = false) LocalDate from,
        @RequestParam(required = false) LocalDate to) {
    return ResponseEntity.ok(eventService.getEvents(from, to));
}
```

## Verificacion

- Existe estructura modular del modulo Event.
- Los endpoints CRUD y de filtro responden desde el modulo nuevo.
- No se depende de controladores o servicios legacy para devolver eventos.
- La suite del modulo y el guardrail modular pasan.

## Resultado esperado

Event vuelve a estar disponible como parte del backend, pero ahora bajo arquitectura modular y con validaciones compatibles con el resto del sistema.

## Criterio de cierre

- [ ] Existe modulo `event` con capas completas.
- [ ] CRUD y filtros de rango estan implementados.
- [ ] La vinculacion con materias no usa imports directos.
- [ ] Los tests del modulo existen y pasan.
- [ ] Ya no hace falta depender del `410 GONE` legacy para este dominio.

## Troubleshooting breve

No reemplazar el `410 GONE` por logica parcial. El legacy debe seguir apagado hasta que el modulo nuevo cubra el contrato necesario.
