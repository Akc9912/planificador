# 1.6 Comunicacion Entre Modulos

## Objetivo

Establecer y verificar que la colaboracion entre modulos se hace solo mediante interfaces compartidas y eventos de dominio, sin imports directos entre modulos y sin dependencia de capas legacy desde codigo nuevo.

## Por que se hace

La arquitectura modular pierde valor en el instante en que un modulo importa la implementacion interna de otro. Este punto es el guardrail que evita reconstruir un monolito acoplado bajo otra carpeta.

## Pasos de ejecucion

1. Revisar las interfaces disponibles en `shared/api`.
2. Revisar eventos disponibles en `shared/event`.
3. Confirmar que los modulos activos implementan o consumen esos contratos.
4. Ejecutar el test de arquitectura para bloquear imports indebidos.
5. Corregir cualquier import directo entre modulos o desde capas legacy.

## Comandos exactos

```bash
find src/main/java/aktech/planificador/shared -type f | sort
grep -R "implements CareerApi\|ApplicationEventPublisher\|@EventListener" -n src/main/java/aktech/planificador
sed -n '1,220p' src/test/java/aktech/planificador/architecture/ModuleBoundariesTest.java
./mvnw -q -DskipTests=false -Dtest=ModuleBoundariesTest test
```

## Ejemplo de codigo

```java
public interface CareerApi {
    boolean userOwnsCareer(UUID userId, UUID careerId);
}

@Service
public class CareerService implements CareerApi {
}
```

## Verificacion

- Existen contratos en `shared/api` y/o eventos en `shared/event`.
- Los modulos no se importan directamente entre si.
- El test `ModuleBoundariesTest` pasa.
- El codigo nuevo no importa `Controller`, `Service`, `Repository`, `Model`, `Dto`, `security` o `Exception` legacy.

## Resultado esperado

La arquitectura queda protegida por convencion y por automatizacion, reduciendo el riesgo de mezclar el avance modular con estructuras anteriores.

## Criterio de cierre

- [x] Cada dependencia entre modulos se resuelve por contrato o evento.
- [x] No hay imports directos modulo a modulo.
- [x] No hay imports desde capas legacy en codigo modular nuevo.
- [x] `ModuleBoundariesTest` queda en verde.
