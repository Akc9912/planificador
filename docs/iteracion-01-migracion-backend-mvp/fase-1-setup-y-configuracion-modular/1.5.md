# 1.5 APIs REST Basicas

## Objetivo

Dejar definidos y verificables los endpoints CRUD base del backend, con autenticacion, ownership, DTOs de entrada y salida, y manejo consistente de errores para las entidades del MVP.

## Por que se hace

Sin APIs basicas cerradas no existe contrato estable para backend ni base reutilizable para el trabajo posterior de frontend o de modulos dependientes.

## Pasos de ejecucion

1. Revisar controladores y DTOs existentes en codigo modular y legacy aun activo.
2. Confirmar que cada endpoint sensible exige autenticacion.
3. Verificar que el `userId` se toma del contexto de seguridad y no del request cuando aplique.
4. Levantar la aplicacion para probar al menos un endpoint real.
5. Documentar el resultado de las pruebas sobre endpoints prioritarios.

## Comandos exactos

```bash
grep -R "@RequestMapping\|@GetMapping\|@PostMapping\|@PutMapping\|@DeleteMapping" -n src/main/java/aktech/planificador/modules src/main/java/aktech/planificador/Controller
./mvnw -q -DskipTests compile
export TOKEN="reemplazar-por-jwt-valido"
./mvnw spring-boot:run
curl -i -H "Authorization: Bearer $TOKEN" http://localhost:8081/api/careers
```

## Ejemplo de codigo

```java
@GetMapping("/api/careers")
public ResponseEntity<List<CareerResponseDto>> getCareers(Authentication authentication) {
    UUID userId = (UUID) authentication.getPrincipal();
    return ResponseEntity.ok(careerService.getCareersByUser(userId));
}
```

## Verificacion

- Existen endpoints base para los modulos activos del MVP.
- Un request autenticado devuelve respuesta valida.
- El endpoint no recibe `userId` por query o path cuando la identidad debe salir del token.
- Los DTOs separan request y response.

## Resultado esperado

El backend expone una base REST minima, autenticada y coherente, util para seguir cerrando Career y para habilitar el resto del MVP.

## Criterio de cierre

- [ ] Los endpoints prioritarios estan identificados y probados.
- [ ] La identidad del usuario sale del token cuando corresponde.
- [ ] Hay DTOs de request y response en el codigo nuevo.
- [ ] El endpoint de ejemplo responde con autenticacion valida.

## Troubleshooting breve

Si `curl` devuelve `401`, validar primero el token y la carga del `SecurityContext`. Si devuelve `403`, revisar reglas de autorizacion antes de asumir fallo del controlador.
