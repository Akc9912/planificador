# 1.7 Testing

## Objetivo

Definir y ejecutar la suite minima de pruebas para la base modular, cubriendo auth, career y reglas arquitectonicas sin agregar nuevas pruebas al codigo legacy.

## Por que se hace

El proyecto necesita un piso tecnico de confianza antes de avanzar a Subject y Equivalence. Sin ese piso, cada paso posterior se apoya en supuestos no verificados.

## Pasos de ejecucion

1. Ejecutar la suite modular actual.
2. Confirmar que los tests cubren auth, career y guardrails.
3. Revisar que no se hayan agregado pruebas nuevas al codigo legacy.
4. Documentar resultados y dejar la orden de ejecucion lista para CI o reproduccion local.

## Comandos exactos

```bash
./mvnw -q -DskipTests=false -Dtest=ModuleBoundariesTest,CareerServiceTest,CareerControllerTest,AuthSessionServiceTest,AuthJwtAuthenticationFilterTest,AuthModuleControllerTest,SecurityConfigAuthIntegrationTest test
./mvnw -q -DskipTests compile
```
