# 1.2 Configurar Conexion a Base de Datos

## Objetivo

Dejar documentada y verificable la configuracion del datasource PostgreSQL para trabajar contra el esquema objetivo definido por el proyecto, usando variables de entorno y sin acoplar el backend a credenciales hardcodeadas.

## Por que se hace

El backend depende de PostgreSQL y de la estructura definida en `humanis_db_init.sql`. Si la configuracion de datasource queda ambigua, cualquier colaborador nuevo pierde tiempo o corre pruebas sobre una configuracion inconsistente.

## Pasos de ejecucion

1. Revisar el archivo de propiedades principal.
2. Confirmar que el driver PostgreSQL esta declarado en Maven.
3. Verificar que la aplicacion usa variables de entorno para host, puerto, nombre, usuario y password.
4. Compilar para validar que la configuracion no rompe el build.

## Comandos exactos

```bash
sed -n '1,120p' src/main/resources/application.properties
grep -n "postgresql\|spring.datasource\|PLANNI_DB" src/main/resources/application.properties
grep -n "<artifactId>postgresql</artifactId>" pom.xml
./mvnw -q -DskipTests compile
```

## Ejemplo de codigo

```properties
spring.datasource.url=jdbc:postgresql://${PLANNI_DB_HOST}:${PLANNI_DB_PORT}/${PLANNI_DB_NAME}
spring.datasource.username=${PLANNI_DB_USER}
spring.datasource.password=${PLANNI_DB_PASSWORD}
```

## Verificacion

- El `application.properties` referencia PostgreSQL.
- No hay credenciales reales hardcodeadas en el repo.
- El `pom.xml` incluye el driver `postgresql`.
- El proyecto sigue compilando despues de revisar la configuracion.

## Resultado esperado

La configuracion de base de datos queda clara, portable entre ambientes y lista para usarse con el schema objetivo del backend.

## Criterio de cierre

- [x] El datasource usa variables de entorno.
- [x] El driver PostgreSQL esta declarado.
- [x] El archivo de propiedades tiene host, puerto y base parametrizados.
- [x] El build sigue en verde.

## Troubleshooting breve

La compilacion puede pasar aunque falten variables de entorno. El fallo real aparece al levantar la aplicacion. Si ocurre, revisar primero las variables `PLANNI_DB_*` del entorno activo.
