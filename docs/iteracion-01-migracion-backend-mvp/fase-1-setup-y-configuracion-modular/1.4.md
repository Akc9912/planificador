# 1.4 Configurar Validacion JWT Supabase

## Objetivo

Dejar estable y verificable la validacion de JWT emitidos por Supabase dentro del backend modular, extrayendo el `userId` desde el token y cargando el `SecurityContext` sin depender de una tabla `users` propia.

## Por que se hace

La autenticacion es la puerta de entrada del backend. Si el backend no valida correctamente el JWT o mezcla auth nueva con auth legacy, toda la seguridad y el control de ownership quedan comprometidos.

## Pasos de ejecucion

1. Revisar propiedades de JWT y secreto Supabase.
2. Revisar el modulo `auth` y la configuracion global de seguridad.
3. Confirmar que el filtro JWT modular es el unico activo para auth nueva.
4. Ejecutar la suite de pruebas de seguridad y auth modular.
5. Verificar que login, register y change-password siguen delegados a Supabase cuando corresponda.

## Comandos exactos

```bash
grep -R "jwt.secret\|supabase.jwt.secret" -n src/main/resources
find src/main/java/aktech/planificador/modules/auth -type f | sort
sed -n '1,240p' src/main/java/aktech/planificador/Config/SecurityConfig.java
./mvnw -q -DskipTests=false -Dtest=AuthSessionServiceTest,AuthJwtAuthenticationFilterTest,AuthModuleControllerTest,SecurityConfigAuthIntegrationTest test
```

## Ejemplo de codigo

```java
String token = authorizationHeader.substring(7);
UUID userId = jwtService.extractUserId(token);
UsernamePasswordAuthenticationToken authentication =
        new UsernamePasswordAuthenticationToken(userId, null, authorities);
SecurityContextHolder.getContext().setAuthentication(authentication);
```

## Verificacion

- Existe configuracion de secretos JWT en propiedades.
- El filtro modular auth se encuentra bajo `modules/auth`.
- La seguridad carga autenticacion a partir del token.
- La suite de auth y seguridad termina en verde.

## Resultado esperado

El backend acepta tokens validos de Supabase, rechaza tokens invalidos y expone el `userId` autenticado para reglas de ownership y autorizacion.

## Criterio de cierre

- [x] La validacion JWT depende del modulo auth actual.
- [x] No quedan clases legacy activas para auth nueva.
- [x] El `SecurityContext` recibe `userId` desde el token.
- [x] Las pruebas de auth y seguridad pasan.
