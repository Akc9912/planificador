# 1.1 Crear Proyecto Spring Boot

## Objetivo

Validar y dejar firme la base del proyecto Spring Boot 3.5.4 con Maven, estructura modular por dominio y convenciones de carpetas que permitan seguir migrando sin mezclar codigo nuevo con capas legacy.

## Por que se hace

Sin una base de proyecto consistente no hay forma confiable de compilar, probar, aislar modulos ni detectar regresiones cuando entren Subject, Equivalence y los modulos posteriores.

## Pasos de ejecucion

1. Verificar `pom.xml` y confirmar dependencias base del backend.
2. Verificar la clase principal y el paquete raiz del proyecto.
3. Confirmar que existen las carpetas de `shared` y `modules` bajo la raiz Java.
4. Compilar sin tests para comprobar que la base tecnica actual sigue sana.

## Comandos exactos

```bash
pwd
sed -n '1,220p' pom.xml
sed -n '1,80p' src/main/java/aktech/planificador/PlanificadorApplication.java
find src/main/java/aktech/planificador -maxdepth 2 -type d | sort
./mvnw -q -DskipTests compile
```

## Ejemplo de codigo

```java
package aktech.planificador;

import org.springframework.boot.autoconfigure.SpringBootApplication;

@SpringBootApplication
public class PlanificadorApplication {
}
```

## Verificacion

- `pom.xml` declara Spring Boot, Web, Security, Data JPA y PostgreSQL.
- Existe `PlanificadorApplication` en el paquete raiz.
- Existen carpetas equivalentes a `shared/` y `modules/` dentro de `src/main/java/aktech/planificador/`.
- La compilacion termina sin errores.

## Resultado esperado

El proyecto queda identificado como la base oficial del backend MVP modular y puede usarse como punto de partida para las siguientes tareas sin reestructuracion adicional.
