# 4.3 Hardening de Seguridad

## Objetivo

Completar el endurecimiento de seguridad con validacion de inputs, reglas por rol, headers de seguridad, CORS controlado y superficie minima de exposicion.

## Por que se hace

La seguridad basica de JWT y roles no alcanza para produccion. Esta tarea reduce errores evitables y baja el riesgo de exponer el backend con defaults inseguros.

## Pasos de ejecucion

1. Revisar `SecurityConfig` y reglas por endpoint.
2. Agregar o verificar `@Valid` en requests y DTOs.
3. Definir CORS segun frontend autorizado.
4. Configurar headers de seguridad y negar rutas innecesarias.
5. Validar comportamiento con pruebas de seguridad.

## Comandos exactos

```bash
sed -n '1,260p' src/main/java/aktech/planificador/Config/SecurityConfig.java
grep -R "@Valid\|@NotNull\|@NotBlank\|@Size" -n src/main/java/aktech/planificador
grep -R "cors\|frameOptions\|contentTypeOptions\|csrf" -n src/main/java/aktech/planificador
./mvnw -q -DskipTests=false -Dtest=SecurityConfigAuthIntegrationTest test
```

## Ejemplo de codigo

```java
http
    .headers(headers -> headers
        .frameOptions(frame -> frame.deny())
        .contentTypeOptions(Customizer.withDefaults()))
    .authorizeHttpRequests(auth -> auth
        .requestMatchers("/swagger-ui/**", "/v3/api-docs/**").permitAll());
```

## Verificacion

- Los requests invalidos fallan por validacion y no avanzan a servicio.
- Los endpoints mantienen reglas por rol claras.
- Los headers de seguridad se emiten segun lo configurado.
- La prueba de seguridad sigue pasando o se amplia segun cambios.

## Resultado esperado

El backend queda con una postura de seguridad mas seria y preparada para ambientes menos controlados que desarrollo local.

## Criterio de cierre

- [ ] DTOs y endpoints sensibles validan input.
- [ ] Reglas por rol estan verificadas.
- [ ] CORS y headers estan definidos.
- [ ] La prueba de seguridad sigue en verde.

## Troubleshooting breve

Si una regla de seguridad rompe Swagger o el flujo de pruebas, ajustar el matcher con precision. No abrir rutas de mas para resolver rapido un `403`.
