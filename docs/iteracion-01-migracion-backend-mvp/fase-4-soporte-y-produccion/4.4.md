# 4.4 Observabilidad

## Objetivo

Agregar health checks, metricas y base de monitoreo para poder observar el estado del backend antes de considerar una salida a produccion.

## Por que se hace

Sin observabilidad, un backend en produccion solo falla en silencio. Esta tarea hace visibles salud, metricas y comportamiento operativo minimo.

## Pasos de ejecucion

1. Confirmar si existe dependencia de Actuator o agregarla.
2. Exponer endpoints de salud e informacion minima.
3. Preparar metricas para consumo por herramientas de monitoreo.
4. Documentar endpoints y restricciones de acceso.
5. Verificar respuestas locales.

## Comandos exactos

```bash
grep -n "actuator\|micrometer" pom.xml
grep -R "management\." -n src/main/resources
./mvnw -q -DskipTests compile
./mvnw spring-boot:run
curl -i http://localhost:8081/actuator/health
```

## Ejemplo de codigo

```properties
management.endpoints.web.exposure.include=health,info,metrics
management.endpoint.health.show-details=when-authorized
```

## Verificacion

- Existe dependencia o configuracion de observabilidad.
- El endpoint `/actuator/health` responde.
- Las metricas expuestas estan definidas y documentadas.
- La exposicion de endpoints esta controlada.

## Resultado esperado

El backend puede monitorearse de forma basica y entregar señales utiles para soporte operativo y despliegue.

## Criterio de cierre

- [ ] Actuator o equivalente esta configurado.
- [ ] Health check responde correctamente.
- [ ] Existen metricas o exposicion base documentada.
- [ ] La aplicacion sigue compilando y levantando.

## Troubleshooting breve

Si el endpoint de health no responde, revisar primero dependencia de Actuator y configuracion de exposicion antes de tocar seguridad.
