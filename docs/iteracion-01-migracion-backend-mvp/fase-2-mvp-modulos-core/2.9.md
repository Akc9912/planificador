# 2.9 Endpoints de Dashboard

## Objetivo

Exponer endpoints que resuman el estado academico del usuario para la vista principal: recomendaciones, materias activas, estadisticas y alertas utiles.

## Por que se hace

El dashboard concentra el valor visible del backend. Esta tarea convierte calculos y estados internos en un contrato de lectura listo para consumo de interfaz, sin mover logica compleja al frontend.

## Pasos de ejecucion

1. Definir el DTO de respuesta del dashboard.
2. Reusar progreso, correlativas y estado de materias como insumos.
3. Implementar servicio agregador del dashboard.
4. Exponer endpoint y documentar response.
5. Agregar pruebas del agregador.

## Comandos exactos

```bash
grep -R "CareerDashboardResponseDto\|SubjectDashboardService\|/career/{careerId}/dashboard" -n src/main/java/aktech/planificador/modules/subject
./mvnw -q -DskipTests compile
./mvnw -q -DskipTests=false -Dtest=SubjectServiceTest,SubjectControllerTest,SubjectDashboardServiceTest,ModuleBoundariesTest test
```

## Ejemplo de codigo

```java
public CareerDashboardResponseDto getCareerDashboard(UUID userId, UUID careerId) {
                return subjectDashboardService.getCareerDashboard(userId, careerId);
}
```

## Contrato implementado en Subject

- `GET /subjects/career/{careerId}/dashboard`
        - Devuelve resumen academico de la carrera para la vista principal.
        - Incluye `currentSubjects`, `recommendedSubjects`, `progress`, `activeSubjects`, `weeklyHours` y `alerts`.
        - DTO principal: `CareerDashboardResponseDto`.
        - Servicio agregador: `SubjectDashboardService`.

## Campos principales de respuesta

- `careerId`: identificador de carrera.
- `currentSubjects`: materias en estado `cursando`.
- `recommendedSubjects`: materias `pendiente` habilitadas por correlativas.
- `progress`: snapshot de progreso reutilizando `CareerProgressResponseDto` (2.6).
- `activeSubjects`: cantidad de materias activas.
- `weeklyHours`: suma de horas de materias activas.
- `alerts`: mensajes utiles de estado (ej.: carrera sin materias, materias bloqueadas, sin activas).
