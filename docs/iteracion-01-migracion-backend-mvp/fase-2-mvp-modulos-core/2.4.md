# 2.4 Implementar Auth Module

## Objetivo

Consolidar el modulo `auth` como unico punto de autenticacion modular del backend, manteniendo validacion JWT, reglas de seguridad y contratos listos para handoff con frontend cuando corresponda.

## Por que se hace

Auth ya tiene avance real. Esta tarea no es rehacerlo, sino estabilizarlo y dejarlo documentado como modulo cerrado para no reabrir discusiones cuando entren los modulos restantes.

## Pasos de ejecucion

1. Auditar clases activas del modulo auth.
2. Confirmar que login, register y cambio de password siguen delegados a Supabase cuando aplique.
3. Confirmar que no hay clases legacy de auth reintroducidas.
4. Ejecutar la suite de auth y seguridad.
5. Dejar documentado el contrato que consumira frontend cuando toque su migracion.

## Comandos exactos

```bash
find src/main/java/aktech/planificador/modules/auth -type f | sort
find src/test/java/aktech/planificador/modules/auth -type f | sort
./mvnw -q -DskipTests=false -Dtest=AuthSessionServiceTest,AuthJwtAuthenticationFilterTest,AuthModuleControllerTest,SecurityConfigAuthIntegrationTest test
grep -R "Supabase\|410 GONE\|change-password\|login\|register" -n src/main/java/aktech/planificador/modules/auth src/main/java/aktech/planificador/Config
```

## Ejemplo de codigo

```java
@PostMapping("/api/auth/validate-token")
public ResponseEntity<LoginResponseDto> validateToken(@RequestBody TokenValidationRequestDto request) {
    return ResponseEntity.ok(authSessionService.validateToken(request));
}
```

## Verificacion

- Existe una sola implementacion auth vigente para codigo modular.
- Las pruebas del modulo auth quedan en verde.
- Las reglas de seguridad siguen aplicadas.
- El contrato auth queda listo para ser consumido por frontend cuando toque su migracion.

## Resultado esperado

Auth queda cerrado, estable y sin mezcla con codigo legacy, listo para acompañar el resto del MVP sin convertirse en fuente de regresiones.

## Criterio de cierre

- [ ] Solo queda activa la implementacion modular de auth.
- [ ] La suite de auth y seguridad pasa en verde.
- [ ] El contrato de endpoints auth esta claro.
- [ ] No se reintroduce codigo legacy en auth.

## Troubleshooting breve

Si aparece una necesidad nueva de auth, agregarla al modulo `modules/auth`. No revivir controladores o servicios legacy como solucion rapida.
