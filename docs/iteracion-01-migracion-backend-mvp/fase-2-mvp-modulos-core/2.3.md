# 2.3 Implementar Equivalence Module

## Objetivo

Implementar el modulo `equivalence` para gestionar equivalencias entre materias, validando reglas de negocio y evitando ciclos o relaciones inconsistentes.

## Por que se hace

Las equivalencias afectan disponibilidad, progreso y consistencia del plan academico. Si este modulo nace acoplado a Subject o Career por implementacion concreta, el costo de mantenimiento se dispara.

## Pasos de ejecucion

1. Crear o completar la estructura `modules/equivalence`.
2. Modelar equivalencias con UUID y ownership.
3. Resolver dependencias con Subject y Career via contratos compartidos o eventos.
4. Implementar CRUD y consultas por materia.
5. Validar no circularidad y consistencia por carrera.
6. Agregar pruebas propias del modulo.

## Comandos exactos

```bash
find src/main/java/aktech/planificador/modules/equivalence -type f | sort
grep -R "Equivalence\|SubjectApi\|CareerApi" -n src/main/java/aktech/planificador/modules/equivalence src/main/java/aktech/planificador/shared
./mvnw -q -DskipTests compile
./mvnw -q -DskipTests=false -Dtest=Equivalence*Test,ModuleBoundariesTest test
```

## Ejemplo de codigo

```java
public void validateNoCycle(UUID sourceSubjectId, UUID targetSubjectId) {
    if (sourceSubjectId.equals(targetSubjectId)) {
        throw new BusinessException("A subject cannot be equivalent to itself");
    }
}
```

## Verificacion

- El modulo compila con capas completas.
- Existen reglas contra equivalencias circulares.
- El modulo consulta otros dominios via contratos, no via servicios concretos.
- La suite del modulo y el test de arquitectura pasan.

## Resultado esperado

El backend puede gestionar equivalencias de forma segura y coherente con las reglas academicas del MVP.

## Criterio de cierre

- [ ] CRUD de equivalencias implementado.
- [ ] Validacion de circularidad implementada.
- [ ] Dependencias cruzadas resueltas con contratos compartidos.
- [ ] El modulo tiene pruebas propias.
- [ ] No existe acoplamiento nuevo con legacy.

## Troubleshooting breve

Si la validacion de equivalencias requiere informacion de Subject, exponer el dato por `shared/api` y no por import directo a `SubjectService`.
