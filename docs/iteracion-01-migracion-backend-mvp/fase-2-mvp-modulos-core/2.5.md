# 2.5 Validacion de Correlativas

## Objetivo

Implementar la logica que determina si una materia esta bloqueada o disponible en funcion de sus correlativas, y calcular que materias se habilitan cuando cambia el estado de una materia previa.

## Por que se hace

Esta es una de las reglas de negocio mas importantes del producto. Si falla, el backend entrega recomendaciones, estados y permisos academicos incorrectos.

## Pasos de ejecucion

1. Definir donde vive la regla dentro del modulo Subject.
2. Modelar la consulta de correlativas y estados necesarios.
3. Implementar validacion de materia bloqueada o disponible.
4. Implementar calculo de materias habilitadas por cambio de estado.
5. Agregar pruebas con casos positivos, negativos y borde.

## Comandos exactos

```bash
grep -R "correlative\|availability\|unlocks" -n src/main/java/aktech/planificador/modules/subject
./mvnw -q -DskipTests compile
./mvnw -q -DskipTests=false -Dtest=SubjectServiceTest,SubjectControllerTest,ModuleBoundariesTest test
```

## Ejemplo de codigo

```java
boolean blocked = toCorrelativeList(subject.getCorrelatives()).stream()
        .anyMatch(correlativeId -> !isCorrelativeApproved(correlativeId, subjectsById, overrideSubjectId, overrideStatus));
```

## Contrato implementado en Subject

- `GET /subjects/{id}/availability`
  - Responde disponibilidad academica real de la materia segun correlativas.
  - Fuente de verdad: `SubjectService#getAvailability`.
- `GET /subjects/{id}/unlocks/{status}`
  - Simula cambio de estado de una materia previa y devuelve materias que se habilitan por ese cambio.
  - Fuente de verdad: `SubjectService#listUnlockedByStatusChange`.

## Verificacion

- La materia queda marcada como bloqueada si alguna correlativa no esta aprobada.
- La materia queda disponible si no tiene correlativas o si todas estan aprobadas.
- Existen pruebas que cubren ambos caminos.
