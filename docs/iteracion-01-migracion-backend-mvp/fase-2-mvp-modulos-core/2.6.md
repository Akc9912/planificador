# 2.6 Calculo de Progreso

## Objetivo

Implementar el calculo de progreso academico por carrera, incluyendo materias aprobadas, pendientes, cursando, creditos acumulados, porcentaje de avance y estadisticas derivadas.

## Por que se hace

El progreso es uno de los principales valores del backend frente al frontend y frente al uso directo de Supabase. Sin esta logica, el MVP queda como CRUD con poco valor agregado.

## Pasos de ejecucion

1. Definir el DTO de progreso y el servicio responsable.
2. Calcular metricas por cantidad de materias y por creditos, si aplica.
3. Reusar el estado de correlativas y materias aprobadas como base.
4. Exponer endpoint de progreso y estadisticas.
5. Agregar pruebas de calculo.

## Comandos exactos

```bash
grep -R "progress\|stats\|DashboardDataDto\|CareerBasicDto" -n src/main/java/aktech/planificador
./mvnw -q -DskipTests compile
./mvnw -q -DskipTests=false -Dtest=CareerServiceTest,SubjectServiceTest test
```

double progressPercentage = totalSubjects == 0

## Verificacion

- El backend puede devolver porcentaje y contadores de avance.
- Los calculos no dependen de clases legacy.
- Existen pruebas con carreras vacias y con datos parciales.

## Resultado esperado

La API puede responder progreso real por carrera y alimentar vistas de resumen sin calculos duplicados en frontend.

## Criterio de cierre

- [ ] Existe DTO de progreso.
- [ ] Existe servicio de calculo.
- [ ] Existe endpoint asociado.
- [ ] Hay pruebas de escenarios basicos y borde.

## Troubleshooting breve

Si aparece una metrica nueva, agregarla en el servicio de progreso. No repartir calculos parciales en varios controladores porque eso rompe consistencia.
