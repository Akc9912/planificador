# 3.2 - Migrar Auth a capas target

## Objetivo

Alinear Auth a la arquitectura de capas objetivo, manteniendolo como modulo de soporte transversal.

## Razon Tecnica

Auth debe permanecer desacoplado del dominio academico, con responsabilidades claras de seguridad/sesion dentro del marco arquitectonico definido.

## Comandos Exactos

```bash
# Crear estructura de capas target para Auth
mkdir -p src/main/java/aktech/planificador/modules/auth/presentation
mkdir -p src/main/java/aktech/planificador/modules/auth/application
mkdir -p src/main/java/aktech/planificador/modules/auth/domain
mkdir -p src/main/java/aktech/planificador/modules/auth/persistence
mkdir -p src/main/java/aktech/planificador/modules/auth/infrastructure/filter

# Migrar componentes existentes
git mv src/main/java/aktech/planificador/modules/auth/controller \
  src/main/java/aktech/planificador/modules/auth/presentation

git mv src/main/java/aktech/planificador/modules/auth/service \
  src/main/java/aktech/planificador/modules/auth/application

git mv src/main/java/aktech/planificador/modules/auth/filter/AuthJwtAuthenticationFilter.java \
  src/main/java/aktech/planificador/modules/auth/infrastructure/filter/AuthJwtAuthenticationFilter.java

# Validar compilación
./mvnw -q -DskipTests compile

# Validar tests
./mvnw -q -DskipTests=false -Dtest=AuthSessionServiceTest,AuthJwtAuthenticationFilterTest,AuthModuleControllerTest,SecurityConfigAuthIntegrationTest,ModuleBoundariesTest test
```

**Nota:** Toda evidencia de ejecución se registra en el README principal de la iteración.
