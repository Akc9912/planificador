# 3.3 - Consolidar contratos y eventos entre modulos

## Objetivo

Confirmar que la comunicacion entre modulos se mantiene solo por shared/api y shared/event, sin acoplamientos directos.

## Razon Tecnica

La migracion de capas pierde valor si aparece acoplamiento entre modulos durante refactor.

## Comandos Exactos

```bash
mkdir -p docs/iteracion-01-implementacion-arquitectura-stack-backend/evidencias/fase-3

find src/main/java/aktech/planificador/shared/api -maxdepth 1 -name "*.java" | sort \
  > docs/iteracion-01-implementacion-arquitectura-stack-backend/evidencias/fase-3/contratos-shared-api.txt

find src/main/java/aktech/planificador/shared/event -maxdepth 2 -name "*.java" | sort \
  > docs/iteracion-01-implementacion-arquitectura-stack-backend/evidencias/fase-3/eventos-shared.txt

rg -n "import aktech\\.planificador\\.modules\\.[a-z]+\\." src/main/java/aktech/planificador/modules \
  > docs/iteracion-01-implementacion-arquitectura-stack-backend/evidencias/fase-3/imports-cruzados-modulos.txt || true

./mvnw -q -DskipTests=false -Dtest=ModuleBoundariesTest test \
  > docs/iteracion-01-implementacion-arquitectura-stack-backend/evidencias/fase-3/module-boundaries-fase3.log
```

## Ejemplo De Codigo (Si Aplica)

```java
public interface CareerApi {
    boolean userOwnsCareer(UUID userId, UUID careerId);
}
```

## Verificacion

1. Existe inventario actualizado de contratos y eventos compartidos.
2. ModuleBoundariesTest en verde.
3. No aparecen imports directos indebidos entre modulos.

## Resultado Esperado

Reglas de acoplamiento respetadas al cierre de migracion de modulos soporte.

## Criterio De Cierre

- [ ] objetivo cumplido
- [ ] verificacion ejecutada
- [ ] evidencia registrada
- [ ] documentacion relacionada actualizada

## Troubleshooting Breve (Si Aplica)

- Si aparece import cruzado, reemplazar por interfaz en shared/api o evento en shared/event.
