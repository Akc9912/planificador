# 3.1 - Migrar Equivalence a capas target

## Objetivo

Reestructurar el modulo Equivalence en capas target manteniendo reglas de negocio y contratos existentes.

## Razon Tecnica

Equivalence es modulo de consistencia academica; su alineacion cierra la brecha arquitectonica del backend core.

## Comandos Exactos

```bash
# Crear estructura de capas target para Equivalence
mkdir -p src/main/java/aktech/planificador/modules/equivalence/presentation
mkdir -p src/main/java/aktech/planificador/modules/equivalence/application
mkdir -p src/main/java/aktech/planificador/modules/equivalence/domain/model
mkdir -p src/main/java/aktech/planificador/modules/equivalence/persistence
mkdir -p src/main/java/aktech/planificador/modules/equivalence/infrastructure

# Migrar componentes
git mv src/main/java/aktech/planificador/modules/equivalence/controller \
  src/main/java/aktech/planificador/modules/equivalence/presentation

git mv src/main/java/aktech/planificador/modules/equivalence/service \
  src/main/java/aktech/planificador/modules/equivalence/application

git mv src/main/java/aktech/planificador/modules/equivalence/model \
  src/main/java/aktech/planificador/modules/equivalence/domain/model

git mv src/main/java/aktech/planificador/modules/equivalence/repository \
  src/main/java/aktech/planificador/modules/equivalence/persistence

# Validar compilación
./mvnw -q -DskipTests compile

# Validar tests
./mvnw -q -DskipTests=false -Dtest=EquivalenceServiceTest,EquivalenceControllerTest,ModuleBoundariesTest test
```
