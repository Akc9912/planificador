# 1.2 - Estandarizar Java 21 en backend y CI

## Objetivo

Alinear el runtime del backend y el pipeline CI a Java 21 para cumplir el stack base definido.

## Razon Tecnica

Una diferencia de version Java entre entorno local y CI genera fallos de build no deterministas y rompe reproducibilidad.

## Comandos Exactos

```bash
mkdir -p docs/iteracion-01-implementacion-arquitectura-stack-backend/evidencias/fase-1

rg -n "<java.version>|java-version" pom.xml .github/workflows/modular-quality-gates.yml \
  > docs/iteracion-01-implementacion-arquitectura-stack-backend/evidencias/fase-1/java-version-antes.txt

# Actualizar pom.xml a Java 21
# Cambiar <java.version>24</java.version> por <java.version>21</java.version>

# Actualizar workflow CI a Java 21
# Cambiar java-version: "24" por java-version: "21"

./mvnw -v \
  > docs/iteracion-01-implementacion-arquitectura-stack-backend/evidencias/fase-1/maven-version-java21.txt

java -version 2> docs/iteracion-01-implementacion-arquitectura-stack-backend/evidencias/fase-1/java-runtime-local.txt

./mvnw -q -DskipTests compile \
  > docs/iteracion-01-implementacion-arquitectura-stack-backend/evidencias/fase-1/compile-java21.log
```

## Ejemplo De Codigo (Si Aplica)

```xml
<properties>
  <java.version>21</java.version>
</properties>
```

## Verificacion

1. pom.xml define Java 21.
2. workflow de CI define Java 21.
3. La compilacion con Java 21 finaliza en verde.

## Resultado Esperado

Proyecto y CI alineados en Java 21 con build reproducible.
