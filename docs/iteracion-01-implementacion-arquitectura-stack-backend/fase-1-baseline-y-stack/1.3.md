# 1.3 - Docker backend operativo

## Objetivo

Dejar operativa la contenedorizacion del backend como parte del stack base obligatorio.

## Razon Tecnica

Docker garantiza ejecucion consistente entre ambientes y reduce friccion de setup durante la migracion arquitectonica.

## Comandos Exactos

```bash
mkdir -p docs/iteracion-01-implementacion-arquitectura-stack-backend/evidencias/fase-1

# Crear Dockerfile en la raiz del repo
cat > Dockerfile << 'EOF'
FROM eclipse-temurin:21-jre
WORKDIR /app
COPY target/planificador-1.0.0.jar app.jar
EXPOSE 8081
ENTRYPOINT ["java", "-jar", "/app/app.jar"]
EOF

./mvnw -q -DskipTests package \
  > docs/iteracion-01-implementacion-arquitectura-stack-backend/evidencias/fase-1/package-for-docker.log

docker build -t planificador-backend:local . \
  > docs/iteracion-01-implementacion-arquitectura-stack-backend/evidencias/fase-1/docker-build.log

docker run --rm -d --name planificador-backend -p 8081:8081 --env-file .env planificador-backend:local
sleep 8
curl -i http://localhost:8081/v3/api-docs \
  > docs/iteracion-01-implementacion-arquitectura-stack-backend/evidencias/fase-1/docker-smoke-api.log

docker stop planificador-backend
```
