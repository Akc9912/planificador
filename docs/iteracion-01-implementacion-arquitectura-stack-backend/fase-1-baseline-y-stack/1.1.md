# 1.1 - Baseline de arquitectura y contratos

## Objetivo

Congelar una fotografia tecnica del backend actual para comparar, durante la migracion, que se mantiene el comportamiento y se reduce brecha arquitectonica.

## Razon Tecnica

Sin baseline verificable, no se puede demostrar si una refactorizacion mejoro o empeoro la arquitectura y estabilidad del sistema.

## Comandos Exactos

```bash
mkdir -p docs/iteracion-01-implementacion-arquitectura-stack-backend/evidencias/fase-1

find src/main/java/aktech/planificador/modules -maxdepth 3 -type d | sort \
  > docs/iteracion-01-implementacion-arquitectura-stack-backend/evidencias/fase-1/modulos-estructura-actual.txt

find src/main/java/aktech/planificador/shared -maxdepth 3 -type f | sort \
  > docs/iteracion-01-implementacion-arquitectura-stack-backend/evidencias/fase-1/shared-contratos-eventos.txt

./mvnw -q -DskipTests=false -Dtest=ModuleBoundariesTest test \
  > docs/iteracion-01-implementacion-arquitectura-stack-backend/evidencias/fase-1/module-boundaries-baseline.log

./mvnw -q -DskipTests compile \
  > docs/iteracion-01-implementacion-arquitectura-stack-backend/evidencias/fase-1/compile-baseline.log
```
