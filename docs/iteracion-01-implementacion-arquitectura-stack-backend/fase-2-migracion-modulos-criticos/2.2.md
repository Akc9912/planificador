# 2.2 - Migrar Subject a capas target

## Objetivo

Reestructurar Subject en capas target, separando responsabilidades de presentacion, aplicacion, dominio, persistencia e infraestructura.

## Razon Tecnica

Subject concentra la mayor complejidad del dominio academico; su migracion correcta evita deuda tecnica estructural en escalado.

## Comandos Exactos

```bash
mkdir -p docs/iteracion-01-implementacion-arquitectura-stack-backend/evidencias/fase-2

mkdir -p src/main/java/aktech/planificador/modules/subject/presentation
mkdir -p src/main/java/aktech/planificador/modules/subject/application
mkdir -p src/main/java/aktech/planificador/modules/subject/domain/model
mkdir -p src/main/java/aktech/planificador/modules/subject/persistence
mkdir -p src/main/java/aktech/planificador/modules/subject/infrastructure

# Mover grupos completos y ajustar package/imports

git mv src/main/java/aktech/planificador/modules/subject/controller \
  src/main/java/aktech/planificador/modules/subject/presentation

git mv src/main/java/aktech/planificador/modules/subject/service \
  src/main/java/aktech/planificador/modules/subject/application

git mv src/main/java/aktech/planificador/modules/subject/model \
  src/main/java/aktech/planificador/modules/subject/domain/model

git mv src/main/java/aktech/planificador/modules/subject/repository \
  src/main/java/aktech/planificador/modules/subject/persistence

# Listener y adaptadores tecnicos quedan en infrastructure
mkdir -p src/main/java/aktech/planificador/modules/subject/infrastructure/listener

git mv src/main/java/aktech/planificador/modules/subject/listener/CareerEventListener.java \
  src/main/java/aktech/planificador/modules/subject/infrastructure/listener/CareerEventListener.java

./mvnw -q -DskipTests compile

./mvnw -q -DskipTests=false -Dtest=SubjectServiceTest,SubjectControllerTest,SubjectModuleServiceTest,SubjectModuleControllerTest,SubjectScheduleServiceTest,SubjectScheduleControllerTest,SubjectDashboardServiceTest,ModuleBoundariesTest test

# Registrar el resultado en:
# docs/iteracion-01-implementacion-arquitectura-stack-backend/evidencias/fase-2/resumen-fase-2.md
```
