# 2.1 - Migrar Career a capas target

## Objetivo

Reorganizar el modulo Career a la estructura de capas objetivo: presentation, application, domain, persistence, infrastructure.

## Razon Tecnica

Career es el modulo mas critico para ownership y contratos cross-modulo; debe quedar como plantilla arquitectonica para el resto.

## Comandos Exactos

```bash
mkdir -p docs/iteracion-01-implementacion-arquitectura-stack-backend/evidencias/fase-2

mkdir -p src/main/java/aktech/planificador/modules/career/presentation
mkdir -p src/main/java/aktech/planificador/modules/career/application
mkdir -p src/main/java/aktech/planificador/modules/career/domain/model
mkdir -p src/main/java/aktech/planificador/modules/career/persistence
mkdir -p src/main/java/aktech/planificador/modules/career/infrastructure

# Mover componentes actuales hacia capas target
# Ajustar package/imports luego de cada movimiento

git mv src/main/java/aktech/planificador/modules/career/controller/CareerController.java \
  src/main/java/aktech/planificador/modules/career/presentation/CareerController.java

git mv src/main/java/aktech/planificador/modules/career/service/CareerService.java \
  src/main/java/aktech/planificador/modules/career/application/CareerService.java

git mv src/main/java/aktech/planificador/modules/career/model/Career.java \
  src/main/java/aktech/planificador/modules/career/domain/model/Career.java

git mv src/main/java/aktech/planificador/modules/career/repository/CareerRepository.java \
  src/main/java/aktech/planificador/modules/career/persistence/CareerRepository.java

./mvnw -q -DskipTests compile

./mvnw -q -DskipTests=false -Dtest=CareerServiceTest,CareerControllerTest,ModuleBoundariesTest test

# Registrar el resultado en:
# docs/iteracion-01-implementacion-arquitectura-stack-backend/evidencias/fase-2/resumen-fase-2.md
```
