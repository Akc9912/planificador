# 2.3 - Regresion existente de modulos criticos

## Objetivo

Verificar que la migracion arquitectonica de Career y Subject no introdujo regresiones funcionales.

## Razon Tecnica

La fase solo se considera estable si mantiene comportamiento existente con las pruebas actuales en verde.

## Comandos Exactos

```bash
mkdir -p docs/iteracion-01-implementacion-arquitectura-stack-backend/evidencias/fase-2

./mvnw -q -DskipTests compile \
  > docs/iteracion-01-implementacion-arquitectura-stack-backend/evidencias/fase-2/regresion-criticos-compile.log

./mvnw -q -DskipTests=false -Dtest=ModuleBoundariesTest,CareerServiceTest,CareerControllerTest,SubjectServiceTest,SubjectControllerTest,SubjectModuleServiceTest,SubjectModuleControllerTest,SubjectScheduleServiceTest,SubjectScheduleControllerTest,SubjectDashboardServiceTest test \
  > docs/iteracion-01-implementacion-arquitectura-stack-backend/evidencias/fase-2/regresion-criticos-tests.log

rg -n "BUILD SUCCESS|Failures: 0|Errors: 0" \
  docs/iteracion-01-implementacion-arquitectura-stack-backend/evidencias/fase-2/regresion-criticos-tests.log \
  > docs/iteracion-01-implementacion-arquitectura-stack-backend/evidencias/fase-2/regresion-criticos-resumen.txt || true
```
