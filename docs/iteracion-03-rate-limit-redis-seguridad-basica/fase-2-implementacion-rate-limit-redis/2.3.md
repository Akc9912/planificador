# 2.3 - Integrar filtro de rate limit en SecurityFilterChain

## Objetivo

Integrar un filtro de rate limit en la cadena de seguridad para aplicar limites en tiempo real y responder 429 cuando corresponda.

## Razon Tecnica

Sin filtro HTTP, el servicio de rate limit no protege endpoints de abuso.

## Comandos Exactos

```bash
cat src/main/java/aktech/planificador/Config/SecurityConfig.java
cat src/main/java/aktech/planificador/modules/auth/infrastructure/filter/AuthJwtAuthenticationFilter.java
./mvnw -DskipTests compile
```

## Ejemplo De Codigo (Si Aplica)

```txt
Filtro sugerido:
- RateLimitFilter (OncePerRequestFilter)
- obtiene actor por userId (si autenticado) o IP (si anonimo)
- consulta RateLimitService
- si bloquea -> retorna 429 con JSON y Retry-After
```

## Verificacion

1. Filtro se registra en SecurityFilterChain en orden correcto.
2. Endpoints objetivo quedan cubiertos por limites definidos.
3. Respuesta de bloqueo es HTTP 429 y formato JSON uniforme.

## Resultado Esperado

Rate limit funcional en runtime para rutas configuradas.
