# 2.2 - Implementar servicio de rate limit distribuido

## Objetivo

Crear servicio de rate limit basado en Redis que evalua y registra consumo por clave de actor/endpoint.

## Razon Tecnica

Se necesita una implementacion reusable y desacoplada para aplicar limites de forma consistente en filtros de seguridad.

## Comandos Exactos

```bash
rg -n "rate|limit|redis" src/main/java/**/*.java
./mvnw -DskipTests compile
```

## Ejemplo De Codigo (Si Aplica)

```txt
Componentes sugeridos:
- interface RateLimitService
- class RedisRateLimitService

Operacion esperada:
- key = ratelimit:{actor}:{route}:{window}
- incremento atomico en redis
- expiracion por ventana
- decision allow/deny + retryAfterSeconds
```

## Verificacion

1. Servicio devuelve decision de limite de forma deterministica.
2. Clave de Redis contempla actor y endpoint.
3. Se define estrategia de fallback ante fallo de Redis.

## Resultado Esperado

Servicio de rate limit listo para integracion en filtro HTTP.

- si redis responde lento: ajustar timeout y politica de fallback.
