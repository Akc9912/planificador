# 1.3 - Definir configuracion de entorno para redis y seguridad

## Objetivo

Establecer variables de entorno y propiedades requeridas para activar rate limit y controles de seguridad por entorno.

## Razon Tecnica

Sin configuracion externa, los limites quedan hardcodeados y dificiles de ajustar en dev/staging/prod.

## Comandos Exactos

```bash
cat .env.example
cat src/main/resources/application.properties
rg -n "REDIS|RATE_LIMIT|SECURITY|CORS" .env.example src/main/resources/application.properties
```

## Ejemplo De Codigo (Si Aplica)

```txt
Variables sugeridas:
REDIS_HOST=localhost
REDIS_PORT=6379
REDIS_PASSWORD=
RATE_LIMIT_ENABLED=true
RATE_LIMIT_DEFAULT_LIMIT=120
RATE_LIMIT_DEFAULT_WINDOW_SECONDS=60
ALLOWED_ORIGINS=http://localhost:3000
```

## Verificacion

1. .env.example documenta todas las variables nuevas.
2. application.properties consume variables via placeholders.
3. No se exponen secretos reales en repositorio.

## Resultado Esperado

Base de configuracion preparada para implementar rate limit Redis y seguridad basica configurable.
