# 1.2 - Definir politica de limites por endpoint y actor

## Objetivo

Definir matriz inicial de rate limit por tipo de endpoint y tipo de identidad (IP o usuario autenticado).

## Razon Tecnica

Una politica explicita evita decisiones ad hoc y permite ajustar limites de forma controlada segun evidencia.

## Comandos Exactos

```bash
rg -n "@RequestMapping|@GetMapping|@PostMapping|@PutMapping|@DeleteMapping" src/main/java/aktech/planificador/modules/**/presentation/*.java
cat docs/iteracion-03-rate-limit-redis-seguridad-basica/README.md
```

## Ejemplo De Codigo (Si Aplica)

```txt
Matriz inicial sugerida:
- /auth/** publico: 10 req/min por IP
- /careers/admin/**: 30 req/min por usuario
- endpoints autenticados generales: 120 req/min por usuario
- /v3/api-docs/** y /swagger-ui/**: limite estricto en prod
```

## Verificacion

1. Existe matriz documentada con endpoint, actor, ventana y limite.
2. La matriz incluye razon tecnica por grupo de endpoints.
3. La politica tiene criterio de ajuste posterior por metrica.

## Resultado Esperado

Politica de rate limit versionada y lista para implementar en codigo.
