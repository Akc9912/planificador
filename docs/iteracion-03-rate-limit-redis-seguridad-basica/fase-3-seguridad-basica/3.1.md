# 3.1 - Endurecer headers HTTP de seguridad

## Objetivo

Agregar headers de seguridad HTTP basicos para reducir superficie de ataque en respuestas del backend.

## Razon Tecnica

Rate limit no reemplaza controles de cabecera; ambos se complementan para seguridad basica.

## Comandos Exactos

```bash
cat src/main/java/aktech/planificador/Config/SecurityConfig.java
./mvnw -DskipTests compile
```

## Ejemplo De Codigo (Si Aplica)

```txt
Headers sugeridos:
- X-Content-Type-Options: nosniff
- X-Frame-Options: DENY
- Referrer-Policy: no-referrer
- Content-Security-Policy: default-src 'self'
```

## Verificacion

1. SecurityConfig aplica headers de seguridad en respuestas.
2. Endpoints API mantienen compatibilidad funcional.
3. Configuracion por entorno permite ajustes cuando haga falta.

## Resultado Esperado

Cabeceras de seguridad basica activas para backend API.
