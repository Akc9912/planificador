# 3.3 - Reducir fuga de informacion en errores

## Objetivo

Ajustar manejo de excepciones para no exponer detalles internos en respuestas al cliente, manteniendo trazabilidad en logs.

## Razon Tecnica

Mensajes internos en 500 pueden filtrar informacion sensible y facilitar reconocimiento de superficie de ataque.

## Comandos Exactos

```bash
cat src/main/java/aktech/planificador/shared/exception/GlobalExceptionHandler.java
rg -n "handleGeneralException|ResponseEntity<String>|Error interno" src/main/java/aktech/planificador/shared/exception/GlobalExceptionHandler.java
./mvnw -DskipTests=false -Dtest=*ControllerTest test
```

## Ejemplo De Codigo (Si Aplica)

```txt
Respuesta 500 sugerida:
{
  "status": 500,
  "error": "INTERNAL_SERVER_ERROR",
  "message": "Ocurrio un error interno",
  "path": "/subjects"
}
```

## Verificacion

1. Responses 500 no incluyen stack trace ni mensaje tecnico interno.
2. Logs conservan detalle tecnico para diagnostico.
3. Pruebas de controladores en verde con nuevo contrato.

## Resultado Esperado

Manejo de errores mas seguro y consistente para produccion.
