# 3.2 - Externalizar CORS por entorno

## Objetivo

Sacar lista de origenes permitidos del codigo hardcodeado y moverla a configuracion externa por entorno.

## Razon Tecnica

CORS hardcodeado dificulta despliegue multi-entorno y aumenta riesgo de cambios manuales inseguros.

## Comandos Exactos

```bash
cat src/main/java/aktech/planificador/Config/SecurityConfig.java
cat src/main/resources/application.properties
cat .env.example
./mvnw -DskipTests compile
```

## Ejemplo De Codigo (Si Aplica)

```txt
application.properties
app.security.allowed-origins=${ALLOWED_ORIGINS:http://localhost:3000}

.env.example
ALLOWED_ORIGINS=http://localhost:3000,https://tu-frontend.com
```

## Verificacion

1. SecurityConfig consume origenes desde propiedades.
2. .env.example documenta formato esperado de origenes.
3. Se valida comportamiento en dev y staging.

## Resultado Esperado

Politica CORS parametrizable y consistente con despliegues reales.
