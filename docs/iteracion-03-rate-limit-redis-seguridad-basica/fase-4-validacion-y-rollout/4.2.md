# 4.2 - Actualizar modular quality gates con suite de seguridad

## Objetivo

Incluir pruebas de rate limit y seguridad basica en quality gates para bloquear merges con protecciones rotas.

## Razon Tecnica

Rate limit y seguridad son controles transversales; deben verificarse en CI de forma obligatoria.

## Comandos Exactos

```bash
cat .github/workflows/modular-quality-gates.yml
rg -n "ModuleBoundariesTest|Security|RateLimit" .github/workflows/modular-quality-gates.yml
./mvnw -B -DskipTests=false -Dtest=ModuleBoundariesTest test
./mvnw -B -DskipTests=false -Dtest=*RateLimit*Test,*Security*Test test
```

## Ejemplo De Codigo (Si Aplica)

```txt
Quality gate sugerido:
- compile
- boundaries
- security/rate-limit suite
- suite modular critica
```

## Verificacion

1. Workflow incluye ejecucion de tests de seguridad/rate limit.
2. El gate falla si rompe cualquier control de seguridad clave.
3. Tiempo de pipeline se mantiene razonable para feedback rapido.

## Resultado Esperado

CI con enforcement continuo de controles de seguridad introducidos en iteracion 03.

- si hay tests flaky: aislar dependencia externa y usar doubles.
