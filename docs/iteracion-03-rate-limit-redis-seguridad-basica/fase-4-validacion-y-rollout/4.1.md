# 4.1 - Crear tests de rate limit y respuesta 429

## Objetivo

Cubrir con pruebas automatizadas el comportamiento de rate limit, incluyendo permitidos, bloqueados y codigo 429.

## Razon Tecnica

Sin pruebas especificas, cambios menores en filtros o config pueden desactivar limites sin detectarse.

## Comandos Exactos

```bash
./mvnw -DskipTests=false -Dtest=*RateLimit*Test test
./mvnw -DskipTests=false -Dtest=*Security*Test test
```

## Ejemplo De Codigo (Si Aplica)

```txt
Casos minimos:
- N requests dentro de limite -> 200/401 segun endpoint
- request N+1 -> 429
- reset de ventana -> vuelve a permitir
```

## Verificacion

1. Existen tests unitarios de servicio de rate limit.
2. Existen tests de filtro/controlador que validan 429.
3. Suite de seguridad y rate limit en verde.

## Resultado Esperado

Cobertura automatizada para comportamiento critico de proteccion contra abuso.

## Criterio De Cierre

- [ ] objetivo cumplido
- [ ] verificacion ejecutada
- [ ] evidencia registrada
- [ ] documentacion relacionada actualizada

## Troubleshooting Breve (Si Aplica)

- si tests son inestables por tiempo: mockear reloj o ventana de expiracion.
- si falla aislamiento: limpiar estado Redis entre casos de prueba.
