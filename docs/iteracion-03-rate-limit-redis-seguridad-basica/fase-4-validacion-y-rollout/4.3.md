# 4.3 - Definir rollout gradual y cierre de iteracion 03

## Objetivo

Cerrar iteracion 03 con plan de despliegue progresivo, criterios de rollback y evidencia de validacion completa.

## Razon Tecnica

Cambios de seguridad y limites pueden impactar usuarios reales; el rollout debe ser controlado para minimizar riesgo.

## Comandos Exactos

```bash
./mvnw -DskipTests compile
./mvnw -DskipTests=false test
git status --short
```

## Ejemplo De Codigo (Si Aplica)

```txt
Plan de rollout sugerido:
1) staging con RATE_LIMIT_ENABLED=true y limites conservadores
2) canary en produccion para rutas /auth/**
3) ampliar a rutas autenticadas generales
4) monitorear bloqueos 429 y ajustar umbrales
```

## Verificacion

1. Existe estrategia de rollout por etapas y feature flags.
2. Existe criterio explicito de rollback.
3. README de iteracion y fases actualizado con estado final.

## Resultado Esperado

Iteracion 03 cerrada con seguridad basica activa y despliegue controlado.

## Criterio De Cierre

- [ ] objetivo cumplido
- [ ] verificacion ejecutada
- [ ] evidencia registrada
- [ ] documentacion relacionada actualizada

## Troubleshooting Breve (Si Aplica)

- si aumenta 429 legitimo: ajustar limites por endpoint y actor.
- si impacta experiencia de login: priorizar excepciones controladas para /auth durante canary.
