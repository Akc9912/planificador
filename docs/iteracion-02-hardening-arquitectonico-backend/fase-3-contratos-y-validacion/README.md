# fase-3-contratos-y-validacion

## Contexto De La Fase

Una arquitectura sana necesita contratos de entrada/salida predecibles. Esta fase endurece validacion y errores para reducir bugs funcionales y ambiguedad para clientes API.

## Objetivo De La Fase

Estandarizar validacion de entrada en presentation y contrato de error en toda la API backend.

## Alcance De La Fase

- constraints de Jakarta Validation en DTOs request
- uso de @Valid en controladores
- contrato JSON de errores uniforme
