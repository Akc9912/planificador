# 3.2 - Aplicar @Valid en controllers y manejo de validacion

## Objetivo

Activar ejecucion de constraints Jakarta en endpoints mediante @Valid y controlar errores de validacion de forma uniforme.

## Razon Tecnica

Sin @Valid, las anotaciones en DTO no se ejecutan y se pierde el beneficio de validacion declarativa.

## Comandos Exactos

```bash
rg -n "@RequestBody" src/main/java/aktech/planificador/modules/**/presentation/*.java
rg -n "@Valid" src/main/java/aktech/planificador/modules/**/presentation/*.java
cat src/main/java/aktech/planificador/shared/exception/GlobalExceptionHandler.java
./mvnw -DskipTests=false -Dtest=*ControllerTest test
```

## Ejemplo De Codigo (Si Aplica)

```txt
public CareerResponseDto createCareer(@Valid @RequestBody CareerCreateRequestDto request)
```

## Verificacion

1. Endpoints con body usan @Valid.
2. GlobalExceptionHandler contempla errores de validacion (MethodArgumentNotValidException).
3. Tests de controladores permanecen en verde.

## Resultado Esperado

Validacion de entrada efectivamente aplicada en capa presentation y cubierta por pruebas.
