# 2.1 - Extender ModuleBoundariesTest con reglas por capa

## Objetivo

Agregar reglas automatizadas para dependencias entre capas dentro de cada modulo, ademas de las reglas actuales entre modulos y legacy.

## Razon Tecnica

Los boundaries actuales validan acoplamiento entre modulos, pero no evitan violaciones internas de capa (por ejemplo presentation -> persistence).

## Comandos Exactos

```bash
cat src/test/java/aktech/planificador/architecture/ModuleBoundariesTest.java
rg -n "MODULE_IMPORT_PATTERN|FORBIDDEN_LEGACY_IMPORT_PREFIXES|modulesShouldNotImportOtherModulesDirectly" src/test/java/aktech/planificador/architecture/ModuleBoundariesTest.java
./mvnw -DskipTests=false -Dtest=ModuleBoundariesTest test
```

## Ejemplo De Codigo (Si Aplica)

```txt
Reglas objetivo por capa:
- presentation -> application
- application -> domain|persistence|infrastructure
- domain -> (sin dependencias a spring/persistence/infrastructure)
- persistence -> domain (solo modelo/contratos)
- infrastructure -> application/domain segun adaptador
```

## Verificacion

1. ModuleBoundariesTest falla si existe dependencia de capa prohibida.
2. ModuleBoundariesTest pasa en estado valido.
3. El mensaje de error identifica archivo y regla violada.

## Resultado Esperado

Guardrail de capas ejecutable en local y CI, reduciendo regresiones arquitectonicas.

## Criterio De Cierre

- [ ] objetivo cumplido
- [ ] verificacion ejecutada

- [ ] documentacion relacionada actualizada

## Troubleshooting Breve (Si Aplica)

- si hay muchos falsos positivos: introducir reglas graduales por modulo.
- si aparecen imports ambiguos: reforzar convencion de paquetes por capa.
