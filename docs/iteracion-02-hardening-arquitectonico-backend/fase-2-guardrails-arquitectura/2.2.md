# 2.2 - Integrar guardrails de arquitectura en CI

## Objetivo

Asegurar que las reglas de arquitectura se ejecuten automaticamente en pipeline para evitar que violaciones lleguen a ramas principales.

## Razon Tecnica

Sin enforcement en CI, las reglas dependen de ejecucion manual y la arquitectura se degrada con el tiempo.

## Comandos Exactos

```bash
cat .github/workflows/modular-quality-gates.yml
rg -n "Compile|Enforce Module Boundaries|Run" .github/workflows/modular-quality-gates.yml
./mvnw -B -DskipTests compile
./mvnw -B -DskipTests=false -Dtest=ModuleBoundariesTest test
```

## Ejemplo De Codigo (Si Aplica)

```txt
Workflow esperado:
1) compile
2) module boundaries + reglas por capa
3) suite modular minima (auth/career/subject/equivalence segun alcance)
```

## Verificacion

1. Workflow ejecuta boundaries como paso obligatorio.
2. Pipeline falla cuando se viola una regla de capa.
3. Pipeline mantiene feedback legible para el equipo.

## Resultado Esperado

Guardrails arquitectonicos activos en CI para push y pull request.

## Criterio De Cierre

- [ ] objetivo cumplido

## Troubleshooting Breve (Si Aplica)

- si CI tarda demasiado: separar suite rapida y suite completa por evento.
- si hay flakiness: priorizar pruebas deterministicas en quality gate principal.
