# 4.2 - Introducir mappers explicitos en modulo Subject

## Objetivo

Mover conversiones entity/dto fuera de SubjectService hacia mappers explicitos para mejorar separacion de responsabilidades.

## Razon Tecnica

Mapeos embebidos en servicios aumentan ruido y dificultan evolucion de contratos de API.

## Comandos Exactos

```bash
rg -n "toResponseDto\(|set[A-Za-z]+\(" src/main/java/aktech/planificador/modules/subject/application/SubjectService.java
./mvnw -DskipTests compile
./mvnw -DskipTests=false -Dtest=SubjectServiceTest,SubjectControllerTest test
```

## Ejemplo De Codigo (Si Aplica)

```txt
Nuevo mapper sugerido:
modules/subject/infrastructure/mapper/SubjectMapper.java

Uso:
- SubjectService delega conversion Subject -> SubjectResponseDto
```

## Verificacion

1. SubjectService delega conversiones a mapper dedicado.
2. No se altera contrato de SubjectResponseDto.
3. SubjectServiceTest y SubjectControllerTest en verde.

## Resultado Esperado

Mapeo explicito y reusable, con menor acoplamiento dentro de servicios de aplicacion.
