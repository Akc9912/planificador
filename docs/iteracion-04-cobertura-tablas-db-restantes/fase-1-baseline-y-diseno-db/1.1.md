# 1.1 - Inventariar tablas faltantes y mapa de cobertura

## Objetivo

Construir un inventario verificable de tablas DB faltantes y su traduccion a modulos backend a implementar.

## Razon Tecnica

Sin inventario formal, se corre riesgo de cerrar iteracion sin cubrir todas las tablas requeridas.

## Comandos Exactos

```bash
rg -n "CREATE TABLE public\.events|CREATE TABLE public\.event_schedules|CREATE TABLE public\.event_subjects|CREATE TABLE public\.reminders|CREATE TABLE public\.reminder_subjects|CREATE TABLE public\.user_settings|CREATE TABLE public\.audit_logs|CREATE TABLE public\.support_tickets" humanis_db_init.sql
rg -n "package aktech\.planificador\.modules\.(event|events|reminder|reminders|settings|support|audit)" src/main/java/**/*.java
```

## Ejemplo De Codigo (Si Aplica)

```txt
Matriz tabla -> modulo
- events -> modules/event
- event_schedules -> modules/event
- event_subjects -> modules/event
- reminders -> modules/reminder
- reminder_subjects -> modules/reminder
- user_settings -> modules/settings
- support_tickets -> modules/support
- audit_logs -> modules/audit o shared/audit
```

## Verificacion

1. Existe matriz tabla -> modulo en README de iteracion.
2. Las 8 tablas faltantes estan listadas sin omisiones.
3. Se identifica estado actual: cubierto/no cubierto por tabla.

## Resultado Esperado

Inventario completo de cobertura pendiente de DB y mapeo tecnico para implementacion.
