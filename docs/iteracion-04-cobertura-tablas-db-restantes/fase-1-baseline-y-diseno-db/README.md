# fase-1-baseline-y-diseno-db

## Contexto De La Fase

Antes de implementar nuevos modulos, se requiere alinear el inventario de tablas faltantes, contratos API y reglas de ownership para evitar retrabajo.

## Objetivo De La Fase

Definir baseline tecnico y diseño funcional de la cobertura restante de tablas de base de datos.

## Alcance De La Fase

- inventario tabla por tabla con estado de cobertura actual
- definicion de contratos API iniciales para cada agregado
- reglas de ownership y validaciones base alineadas a RLS
