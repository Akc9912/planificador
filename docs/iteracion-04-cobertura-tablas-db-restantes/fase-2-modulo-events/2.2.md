# 2.2 - Implementar event_schedules

## Objetivo

Implementar gestion de horarios de evento sobre tabla event_schedules vinculada a events.

## Razon Tecnica

Permite modelar ocurrencias y estructura temporal del evento sin sobrecargar entidad principal.

## Comandos Exactos

```bash
rg -n "CREATE TABLE public\.event_schedules|CREATE POLICY .*event_schedules" humanis_db_init.sql
./mvnw -DskipTests=false -Dtest=*EventSchedule*Test test
```

## Ejemplo De Codigo (Si Aplica)

```txt
Flujo sugerido:
POST /events/{eventId}/schedules
GET /events/{eventId}/schedules
DELETE /events/{eventId}/schedules/{scheduleId}
```

## Verificacion

1. No se crean schedules para eventos de otro usuario.
2. Se validan campos temporales requeridos.
3. Pruebas del agregado schedule en verde.

## Resultado Esperado

Subrecurso schedules operativo y consistente con ownership del evento.
