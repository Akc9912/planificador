# 2.3 - Implementar event_subjects con asociacion opcional

## Objetivo

Implementar asociacion opcional entre eventos y materias usando tabla event_subjects.

## Razon Tecnica

El usuario debe poder crear eventos independientes o asociados a una materia existente.

## Comandos Exactos

```bash
rg -n "CREATE TABLE public\.event_subjects|CREATE POLICY .*event_subjects" humanis_db_init.sql
./mvnw -DskipTests=false -Dtest=*EventSubject*Test test
```

## Ejemplo De Codigo (Si Aplica)

```txt
Operaciones sugeridas:
POST /events/{eventId}/subjects/{subjectId}
DELETE /events/{eventId}/subjects/{subjectId}
GET /events/{eventId}/subjects
```

## Verificacion

1. Asociacion solo permitida para recursos del mismo owner.
2. Evento sin subjects sigue siendo valido.
3. Se evita duplicar misma asociacion event-subject.

## Resultado Esperado

Eventos con relacion opcional a subject implementada y validada.

## Troubleshooting Breve (Si Aplica)

- si hay asociaciones huerfanas: validar existencia de event y subject antes de insertar.
- si se repite asociacion: constraint unico por (event_id, subject_id).
